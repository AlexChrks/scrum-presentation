Scrum - agile project management framework
Product backlog:
  • A list of all desired work on the project
  • All requirements are described according to a single template, which is called User Story
  • The requirements are designed in such a way that it is obvious and understandable what value they represent to the user.
  • Prioritized by the product owner
  • Reprioritized at the start of each sprint
Sprint backlog:
  • This is a list of all the work that needs to be done in the next sprint
  • During the sprint, new requirements should not appear in the Sprint backlog.
  • All requirements must be divided into tasks and evaluated
